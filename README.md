# Team AI Strategy

A comprehensive strategy document for team AI enablement, following a layered approach: Standardization → Prompt Enablement → Workflow-oriented Augmentation.

## Overview

This document outlines our approach to applying AI in a scalable, governed, and repeatable way within operations and delivery environments.

## Key Sections

1. **Executive Summary** - Context, motivation, and strategic approach
2. **Ops Work System Overview** - Two interconnected work streams
3. **Ops Workflow & Challenges** - Key activities and pain points
4. **Systemic Friction Patterns** - Recurring operational challenges
5. **From Friction to Enablement** - Layered enablement approach
6. **Execution Plan** - Three-phase implementation roadmap

## Tech Stack

- **Tailwind CSS** - Utility-first CSS framework
- **Mermaid.js** - Diagram and flowchart rendering
- **Vanilla JavaScript** - No framework dependencies

## Deployment

This project is designed to be deployed on Vercel or any static hosting platform.

### Deploy to Vercel

1. Connect your GitHub repository to Vercel
2. Vercel will automatically detect the static site
3. Deploy with default settings

The site will be live at your Vercel domain.

## Local Development

Simply open `index.html` in a web browser, or use a local server:

```bash
# Using Python 3
python3 -m http.server 8000

# Using Node.js (with http-server package)
npx http-server
```

Then visit `http://localhost:8000` in your browser.
